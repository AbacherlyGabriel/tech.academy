Quem Somos?

Nos últimos anos têm crescido a demanda por cursos online. Plataformas como Udemy, Coursera, Alura, entre outras, disponibilizam cursos completos sobre diversos temas e das mais variadas áreas de conhecimento. No atual contexto de isolamento social, muitas dessas plataformas têm oferecido descontos em grande parte dos seus cursos, o que tem favorecido o consumidor. No entanto, a gama de opções de plataformas e cursos exige um filtro sobre qual é a melhor oferta. Dessa forma, nossa proposta é criar uma plataforma onde é possível listar, comparar e avaliar plataformas e cursos online de tecnologia, disponibilizando um filtro para que o usuário possa conhecer a reputação do curso que está considerando adquirir. Além disso, aproveitando o contexto de pandemia, onde muitos trabalhadores têm perdido seus empregos devido aos impactos econômicos da pandemia, negociaríamos com as plataformas um desconto aos usuários que comprovassem terem sido demitidos durante o período de crise atual, servindo como incentivo educacional e dando maiores oportunidades para o reingresso ao mercado de trabalho e estímulo à economia.


Ferramentas

A gestão do projeto está sendo realizada através do Trello, onde especificamos um quadro de acordo com o modelo de Scrum (Backlog, etapas de desenvolvimento, blocks, etc). Para realizar o desenvolvimento, optamos pelo Python no back-end, com o uso de bibliotecas como Flask e PyTest - para realização de testes automatizados. No front, estamos utilizando JS, HTML e CSS, alguns elementos do bootsrap e o figma para prototipação. O banco de dados foi configurado por meio do MongoDB. Como ambientes, estão sendo utilizados o PyCharm e o Visual Studio Code. Além disso, o versionamento do código é feito através do git com o GitHub como repositório. Para a realização do deploy, foi utilizado o Heroku.


Build/Deploy Automatizados

A configuração dos processos de Integração Continua (CI) e Deploy Continuo (CD) foi realizada por meio do GitHub Actions, no qual, podemos acompanhar, passo a passo, o build e o deploy por meio de logs disponibilizados na própria plataforma.


Funções Implementadas

Para essa entrega, implementamos o fluxo de login/cadastro de usuários na plataforma, ou seja, todo o processo que envolve, desde a criação das telas, até as validações no banco de dados e os testes de conexão e criação de novos registros, além das configurações de CI/CD.
